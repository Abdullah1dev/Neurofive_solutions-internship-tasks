# Zero-Shot vs Few-Shot Prompting

A comparative study of **Zero-Shot** and **Few-Shot Prompting** using customer support message classification across multiple Large Language Models (LLMs).

---

## Project Overview

Prompt engineering plays a crucial role in improving the quality and consistency of responses generated by Large Language Models (LLMs). This project compares two fundamental prompting techniques—**Zero-Shot Prompting** and **Few-Shot Prompting**—by evaluating their performance on a customer support message classification task.

The experiment was conducted using three popular LLMs:

- ChatGPT
- Claude
- Gemini

Each model was asked to classify the same set of customer support messages into one of the following categories:

- Complaint
- Question
- Praise

The objective was to analyze whether providing example-based guidance (Few-Shot Prompting) improves the model's classification performance compared to using instructions alone (Zero-Shot Prompting).

---

## Objectives

- Understand the difference between Zero-Shot and Few-Shot Prompting.
- Compare the performance of multiple LLMs using identical datasets.
- Evaluate classification accuracy across different prompting techniques.
- Analyze the impact of prompt design on model performance.

## Methodology

### Step 1: Dataset Preparation

A dataset consisting of **10 customer support messages** was created. Each message was manually assigned a correct category (Ground Truth).

### Step 2: Zero-Shot Prompting

Each model received only task instructions, category definitions, and the customer messages without any examples.

### Step 3: Few-Shot Prompting

The same task was repeated after providing three labeled examples before the actual dataset.

### Step 4: Evaluation

The predictions generated by ChatGPT, Claude, and Gemini were compared against the Ground Truth labels to calculate classification accuracy.

---

## Prompting Techniques

### Zero-Shot Prompting

The model receives only instructions and performs the task without any prior examples.

### Few-Shot Prompting

The model receives task instructions along with a few labeled examples before performing the classification task.

## Key Findings

- All three LLMs successfully classified every customer support message.
- Both prompting techniques achieved **100% accuracy** on the selected dataset.
- The Zero-Shot Prompt was sufficient for this relatively simple classification task.
- Few-Shot Prompting provided additional context through examples and is expected to be more beneficial for complex or ambiguous tasks.

## Conclusion

This project demonstrates how prompt design influences interactions with Large Language Models. While both Zero-Shot and Few-Shot prompting achieved identical performance on this dataset, the experiment highlights the importance of selecting an appropriate prompting strategy based on task complexity. Few-Shot Prompting remains a valuable technique for improving consistency and reducing ambiguity in more challenging real-world applications.
